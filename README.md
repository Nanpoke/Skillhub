# SkillHub 使用说明书

## 产品简介

SkillHub 是一款桌面应用，用于统一管理多个 AI 编程工具的 Skills。

**支持的 AI 工具**：Claude Code、OpenCode、Cursor、CodeBuddy、Trae，并支持**添加自定义工具**

**核心价值**：
- 一处存储，多工具复用
- 一键启用/禁用，秒级同步
- 本地安装、Git 仓库安装
- **自定义分类和标签**，灵活组织管理
- 导入/导出配置，轻松备份迁移

---

## 快速入门

### 第一次启动

1. **配置存储位置** - 选择 SkillHub 数据存储路径（默认 `~/.skill-hub/`）
2. **选择 AI 工具** - 自动检测已安装的工具，或**手动添加自定义工具**
3. **开始使用** - 进入主界面，开始管理你的 Skills

### 日常使用流程

1. **浏览 Skills** - 主界面查看所有已安装的 Skills
2. **安装新 Skills** - 点击右上角"安装"按钮，选择本地安装或 Git 安装
3. **启用/禁用** - 在 Skill 卡片中切换各工具的启用状态
4. **整理分类** - 为 Skill 设置**自定义分类和标签**
5. **查看详情** - 点击查看按钮，浏览 SKILL.md 内容

---

## 核心功能

### Skill 浏览与管理

主界面以卡片列表展示所有 Skills，每个卡片显示：
- **基本信息**：名称、来源、安装时间
- **标签和分类**：自定义的分类和标签
- **工具状态**：显示该 Skill 在哪些工具中已启用

**卡片操作**：
- 👁 查看 - 打开 SKILL.md 查看器
- 🔄 更新 - 从 Git 源更新 Skill
- 🗑 删除 - 移除 Skill 及其文件

**展开详情**：点击展开可编辑分类、添加标签、写入备注，并为每个工具独立设置启用/禁用。

**筛选与搜索**：左侧边栏提供分类筛选、标签筛选、工具筛选；顶部搜索框支持按名称、标签搜索。

---

### 安装 Skills

点击主界面右上角"安装"按钮进入安装页面，支持两种方式：

**本地安装**：
- 拖拽文件夹、.zip 压缩包到上传区域
- 或点击选择本地文件
- 自动扫描并预览可安装的 Skills

**Git 安装**：
- 输入 Git 仓库地址（支持 `user/repo` 简写和完整 URL）
- 支持指定子路径安装
- 解析后预览并选择要安装的 Skills

---

### 设置

点击主界面右上角设置按钮进入设置页面：

**外观设置**
- 主题模式：暗黑 / 亮色(目前还没有优化） / 跟随系统

**工具管理**
- 查看已配置的 AI 工具列表
- 添加自定义工具（指定名称和 Skills 路径）
- 启用/禁用工具的管理

**存储设置**
- 查看当前 SkillHub 存储路径
- 迁移数据到新位置

**数据管理**
- 查看操作历史记录（保留 10 天）
- 导入/导出配置
- 重置所有数据

---

### 导入/导出配置

**导出配置**：
- 将所有 Skills、元数据、工具配置打包为 .zip 文件
- 便于备份或迁移到新设备

**导入配置**：
- 选择 .zip 备份文件恢复数据
- 冲突时可选择"跳过已有"或"覆盖更新"

---

### SKILL.md 查看器

查看 Skill 详情时提供三种视图：
- **文件目录**：浏览 Skill 文件夹结构
- **渲染视图**：Markdown 渲染后的内容
- **代码视图**：查看原始 Markdown 源码

## 技术参考

### 数据存储结构

```
~/.skill-hub/
├── skills/                    # Skill 文件本体
├── config/                    # 各工具启用配置
│   ├── claude-code.json
│   ├── opencode.json
│   └── ...
├── metadata/                  # 元数据（标签、备注）
├── git/                       # Git 仓库缓存
├── history/                   # 操作日志（10天）
├── settings.json              # 应用设置
└── custom-tools.json          # 自定义工具配置
```

### 工具默认路径

| 工具 | Skills 路径 |
|------|-------------|
| Claude Code | `~/.claude/skills/` |
| OpenCode | `~/.config/opencode/skills/` |
| Cursor | `~/.cursor/skills/` |
| CodeBuddy | `~/.codebuddy/skills/` |
| Trae | `~/.trae/skills/` |

### 启用/禁用机制

- **启用**：将 Skill 从 `~/.skill-hub/skills/` 复制到目标工具目录
- **禁用**：删除目标工具目录中的 Skill 文件夹

## 常见问题

**Q: 如何添加不支持的 AI 工具？**
A: 进入设置 → 工具管理 → 添加自定义工具，输入工具名称和其 Skills 存放路径。

**Q: Skill 更新后需要重新启用吗？**
A: 是的，更新会刷新 SkillHub 中的副本，需重新启用才会同步到各工具。

**Q: 为什么有些Skill检测不到更新？**
A: SkillHub支持两种更新检测方式：优先使用GitHub Release的tag版本比较，如果仓库没有发布Release则自动回退到提交时间比较。如果仍然检测不到，可能是GitHub API限流，建议在设置中配置GitHub Token。

**Q: 导入配置会覆盖现有数据吗？**
A: 导入时可选择"跳过已有"或"覆盖更新"，根据需要选择。
